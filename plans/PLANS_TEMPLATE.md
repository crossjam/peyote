# Plan Writing Instructions

* Apply timestamps within a generated or modified plan
* Include checkoff task lists where needed
* Generate a problem description section
  - If there’s a provided github issue or pr, read the issue or pr,
    include a link if possible, and summarize it in the plan
  - If there’s not enough information prompt the user
* Provide a section named "Implementation Approaches"
* Provide a section entitled "Alternative Approaches"
* Provide a section entitled "Implementation Plan"
* Provide a section on "Testing Strategy"
* Provide a section on "Satisfaction Criteria"
* If necessary discuss security and safety
